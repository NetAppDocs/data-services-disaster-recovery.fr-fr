= Restaurez les applications à la source d'origine avec NetApp Disaster Recovery
:hardbreaks:
:allow-uri-read: 
:icons: font
:imagesdir: ../media/use/


[role="lead"]
Une fois une catastrophe résolue, revenez du site de reprise après sinistre au site source pour revenir aux opérations normales.  Vous pouvez sélectionner l'instantané à partir duquel effectuer la récupération.

*Rôle de NetApp Console requis* Rôle d'administrateur d'organisation, d'administrateur de dossier ou de projet, d'administrateur de reprise après sinistre ou d'administrateur de basculement de reprise après sinistre.

link:https://docs.netapp.com/us-en/console-setup-admin/reference-iam-disaster-rec-roles.html["En savoir plus sur les rôles et les autorisations des utilisateurs dans NetApp Disaster Recovery"]. https://docs.netapp.com/us-en/console-setup-admin/reference-iam-predefined-roles.html["En savoir plus sur les rôles d'accès à la NetApp Console pour tous les services"^].



== À propos du failback

En cas de retour en arrière, NetApp Disaster Recovery réplique (resynchronise) toutes les modifications vers la machine virtuelle source d'origine avant d'inverser le sens de la réplication. Ce processus débute à partir d'une relation qui a basculé vers une cible et comprend les étapes suivantes :

* Effectuer un contrôle de conformité sur le site récupéré.
* Actualisez les informations vCenter pour chaque cluster vCenter identifié comme situé sur le site récupéré.
* Sur le site cible, mettez hors tension et désenregistrez les machines virtuelles, puis démontez les volumes.
* Rompre la relation SnapMirror sur la source d'origine pour la rendre en lecture/écriture.
* Resynchronisez la relation SnapMirror pour inverser la réplication.
* Mettez sous tension et enregistrez les machines virtuelles sources, puis montez les volumes sur la source.




== Avant de commencer

Si vous utilisez une protection basée sur un datastore, les machines virtuelles ajoutées au datastore peuvent être ajoutées à ce dernier lors du processus de basculement. Si cela s'est produit, assurez-vous de fournir les informations de mappage supplémentaires pour ces machines virtuelles avant de lancer le basculement. Pour modifier le mappage des ressources, consultez link:manage.html#manage-replication-plans["Gérer les plans de réplication"].



== Étapes

[role="tabbed-block"]
====
.vCenter
--
.Étapes
. Dans le menu NetApp Disaster Recovery , sélectionnez *Plans de réplication*.
. Dans le menu déroulant, définissez le filtre sur les plans de réplication *vCenter*.
+
image::site-type-toggle.png[Capture d'écran du bouton de basculement du type de site]

. Identifiez le plan de réplication que vous souhaitez effectuer un retour arrière. Sélectionnez l'option *Actions* image:../use/icon-horizontal-dots.png["Icône du menu Actions dans le service NetApp Disaster Recovery"], survolez *Récupérer* puis sélectionnez *Retour arrière*.
. Choisissez l'instantané de la banque de données à partir duquel effectuer la restauration. Sélectionnez *Créer un instantané maintenant* pour lancer un nouvel instantané ou *Sélectionner* pour choisir parmi les instantanés disponibles.
+
image:failback.png["Capture d'écran de la boîte de dialogue de restauration"]

. Saisissez « Failback » pour confirmer l'opération, puis sélectionnez *Failback*.


Pour suivre l'avancement de la tâche, sélectionnez *Suivi des tâches* dans le menu Reprise après sinistre.

--
.Kubernetes
--
. Dans le menu NetApp Disaster Recovery , sélectionnez *Plans de réplication*.
. Dans le menu déroulant, filtrez sur les plans de réplication *Kubernetes*.
+
image::site-type-toggle.png[Capture d'écran du bouton de basculement du type de site]

. Identifiez le plan de réplication que vous souhaitez effectuer un retour arrière. Sélectionnez l'option *Actions* image:../use/icon-horizontal-dots.png["Icône du menu Actions dans le service NetApp Disaster Recovery"], survolez *Récupérer* puis sélectionnez *Retour arrière*.
. Choisissez l'instantané de la banque de données à partir duquel effectuer la restauration. Sélectionnez *Prendre un instantané maintenant* pour lancer un nouvel instantané ou *Utiliser le dernier instantané* pour utiliser l'instantané le plus récent et réussi.
. Saisissez « Failback » pour confirmer l'opération, puis sélectionnez *Failback*.


Pour suivre l'avancement de la tâche, sélectionnez *Suivi des tâches* dans le menu Reprise après sinistre.
